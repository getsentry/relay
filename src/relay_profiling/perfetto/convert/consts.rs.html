13/// Maximum number of raw samples we collect from a Perfetto trace before
14/// bailing out. At 100 Hz across multiple threads, a 66-second chunk
15/// produces at most ~6 600 samples per thread; 100 000 provides generous
16/// headroom while bounding memory usage against adversarial input.
17pub const MAX_SAMPLES: usize = 100_000;
18
19/// Maximum number of top-level Perfetto trace packets we decode before
20/// bailing out.
21///
22/// `MAX_SAMPLES` only counts useful `PerfSample` packets. Valid traces also
23/// need clock snapshots, interned data updates, incremental state resets, and
24/// other metadata packets. The extra 10 000 packets are allocation headroom for
25/// that metadata while keeping the top-level packet vector bounded.
26pub const MAX_TRACE_PACKETS: usize = MAX_SAMPLES + 10_000;
27
28/// Maximum encoded size of a single top-level Perfetto trace packet.
29///
30/// This bounds allocations from repeated fields inside an individual packet
31/// before handing the packet body to prost for decoding.
32pub const MAX_TRACE_PACKET_BYTES: usize = 4 * 1024 * 1024;
